How to run the evaluation

Step 1 — Generate: Run the same downstream model and settings twice for each prompt: once with the original prompt and once with the compressed prompt.

Step 2 — Probe: Score both responses against the five questions and required inclusions listed for that prompt. The questions are evaluation probes, not additions to the production prompt.

Step 3 — Score: For each probe use 2 = fully correct and attributable; 1 = partly correct, ambiguous, or missing a qualifier; 0 = incorrect, contradicted, or absent.

Step 4 — Compare: Record original and compressed scores separately. A compression-attributable loss is present when the compressed result scores lower and the missing element is available in the original prompt.

Recommended gate: Fail a prompt if any critical probe drops below 2, any numeric value is attached to the wrong label, an instruction/negation/override changes meaning, or the compressed total falls more than 10% below the original total.

Loss taxonomy

Instruction/decision scope: Changes what the model is asked to decide or how it must respond.

Label/relationship: Keeps a value but removes the name or relationship needed to interpret it.

Qualifier/evidence: Drops words such as primarily, very, few, only, before, or after that affect strength or meaning.

Numeric/temporal: Changes signs, thresholds, counts, dates, intervals, or units.

Availability/negation: Changes absent, unavailable, none, deferred, or do-not-use semantics.

Provenance/identifier: Corrupts paths, URLs, model names, issue IDs, authors, or repository names.

Output contract: Weakens JSON, schema, length, range, or formatting requirements.

Cross-prompt conclusions

Most serious failure mode: Label deletion. Prompt 1 demonstrates that preserving a number without its field name can be worse than deleting the number because it invites confident misattribution.

Second serious failure mode: Qualifier deletion. Prompt 3 removes evidence-strength words central to the requested AI-authorship judgment.

Context matters: Prompt 5's provenance damage is material for auditing but probably immaterial for a 1-3 sentence feature summary. Fidelity gates should therefore combine universal checks with task-specific probes.

What survived well: Thresholds, schedules, channel overrides, explicit absences, and major implementation requirements were generally preserved in Prompts 2, 4, and 5.

Recommended benchmark metric: Report critical-probe pass rate, total probe score retention, numeric-label accuracy, instruction/negation accuracy, and identifier exact-match rate alongside token savings.
